# Capacitor bootstrap (run once on a fresh clone)

The native folders (`android/`, `ios/`) are gitignored тАФ they're regenerated by
the Capacitor CLI using local SDKs that aren't suitable to commit. After
pulling, run the steps below. Repeat **Step 3** any time the web bundle changes
(`npm run cap:sync`).

> Buddget is **Egypt-first**. Replace any UAE-only sample copy with EGP / ╪м┘Ж┘К┘З
> defaults before showcasing the app.

## 0. Prerequisites

- Node 20+, npm 10+ (matches the web build).
- **Android**: Android Studio Hedgehog or newer; SDK 34, Build Tools 34.0.0,
  JDK 17. Set `ANDROID_HOME` and have an emulator or attached device.
- **iOS**: macOS with Xcode 16, CocoaPods 1.15+, an active Apple Developer
  team for code-signing.
- Firebase project created at `console.firebase.google.com`. Download
  `google-services.json` (Android) and `GoogleService-Info.plist` (iOS).

## 1. Install + initialise

```bash
npm install
# (capacitor.config.ts is already committed тАФ skip `npx cap init`)
```

## 2. Add native platforms

```bash
npx cap add android
npx cap add ios
```

After `add android`:

- Drop your `google-services.json` into `android/app/`.
- Open `android/app/build.gradle` and apply the Google Services plugin
  (`apply plugin: 'com.google.gms.google-services'`) at the bottom.
- Add the Notification Listener service from `docs/NOTIFICATION_LISTENER.md`
  inside `<application>` of the merged `AndroidManifest.xml`.
- Add the Glance widget receiver from `docs/WIDGET_SETUP.md`.

After `add ios`:

- Drop your `GoogleService-Info.plist` into `ios/App/App/`.
- Open `ios/App/App.xcworkspace`, enable **Push Notifications** + **Background
  Modes (Remote notifications)**, and create the App Group `group.online.buddget`
  on both the main target and the Widget extension.
- Add the WidgetKit extension from `docs/WIDGET_SETUP.md`.

## 3. Build the static web bundle and sync

```bash
NEXT_PUBLIC_API_BASE_URL="https://buddget.app" npm run cap:build
```

This runs `CAPACITOR=true next build --webpack` (output: `out/`) and then
`npx cap sync` so both native projects pick up the new web bundle.

## 4. Run on device / emulator

```bash
npx cap open android
# or
npx cap open ios
```

## 5. Environment variables

Set these in `.env.local` (web dev) and the matching CI secrets:

| Key | Where | Notes |
| --- | ----- | ----- |
| `NEXT_PUBLIC_API_BASE_URL` | Capacitor build only | e.g. `https://buddget.app` |
| `FIREBASE_SERVICE_ACCOUNT_JSON` | Server (Vercel) | full JSON string |
| `NEXT_PUBLIC_FIREBASE_PROJECT_ID` | Web + Capacitor | e.g. `buddget-prod` |
| `NEXT_PUBLIC_FIREBASE_VAPID_KEY` | Web | optional тАФ for FCM JS in WebView |
| `GROQ_API_KEY` | Server | enables `/api/voice/transcribe` |
| `GEMINI_API_KEY` | Server | enables `/api/ai`, `/api/receipt/scan`, `/api/sms/parse` |
| `NOTIFICATION_LISTENER_SHARED_SECRET` | Optional | per-user fallback when a token isn't issued |

Existing Supabase + VAPID keys are unchanged.

## 6. Bumping packages

```bash
npm i @capacitor/core@latest @capacitor/cli@latest
npx cap migrate
```

## 7. Sanity scripts

```bash
npm run cap:sync       # only resync the web bundle
npm run cap:android    # build + open Android Studio
npm run cap:ios        # build + open Xcode
```
