Payment methods · v4 (finalized)
A single bottom sheet opened from a Profile tab. First it shows your saved cards as a carousel (manage each via the on-card ⋯); "Add method" slides in the setup sheet. No dedicated settings page, no list view.
Sheet, not a page. Lives as a tab in the Profile menu; opens as a bottom sheet. View + add/edit are two states of the same sheet.
Manage on the card. A ⋯ on each card opens edit / set default / delete — no action bar, saves space.
Card = type icon + provider name. Top-left glyph is the type (bank / card / wallet…); the provider is the big name. Provider initials (brand-coloured) are used in the pickers.
Cash is implicit. Every user has Cash by default — it never appears as a card, a provider, or a type option.
Rich colour slider + brand default. ~25 colours; the card defaults to the brand's primary and can be recoloured.
Unified currency sheet. The real reused component (flags + full world list) — the same one that replaces every currency dropdown app-wide.
Browse = one pattern. Popular grid + "Search all providers" (no row-vs-dropdown duplication). Only tweak: popularScope user / all.
